常用数组方法补充

Dec 25, 2019

2017年刚工作的时候，总结了数组常用的几种方法，两年过去了，补充一些数组新增的方法。

首先判断是否为数组：Array.isArray()

第一个参数：被转化的数组

第二个参数：对每个元素进行处理，将处理后的结果放入返回的数组（类似于map）

第三个参数：用来绑定this

Array.of()

总会创建一个包含所有传入参数的数组，而不管参数的数量与类型

与Array()相比较：

  1. 当参数等于1时，Array()指的是数组的长度

  2. 当参数大于1时，Array()才会返回由参数组成的数组

  3. Array.of 总是返回参数值组成的数组。如果没有参数，就返回一个空数组。

Array.of基本上可以用来替代Array()或newArray()，并且不存在由于参数不同而导致的重载，而且他们的行为非常统一。

copyWithin()

可以在当前数组内部，将指定位置的数组项复制到其他位置，会覆盖原数组，返回当前数组

参数一：target,从该位置开始替换数据，为负数时倒数（必选）

参数二：start,从该位置开始读取数据，默认为0（可选）

参数三：end,从该位置前停止读取数据，默认为数组长度（可选）

fill()

作用：填充数组

参数一：填充的起始位置(可选)

参数二：填充的结束位置(可选)

find与findIndex

find:返回数组中第一个满足条件的元素，若无，返回undefined
findIndex:返回第一个符合条件的数组成员的位置

上述两种方法的回调函数可以接受的参数：

参数一:当前值 value

参数二:当前位置 index

参数三:原数组 arr

注：

indexOf方法无法识别数组的NaN成员，但是findIndex结合Object.is方法可以做到

includes

判断数组中是否包含给定的值

参数一：需要查找的值

参数二：搜索的起始位置（为-1时，表示从数组的倒数第一位开始）

与indexOf对比

  1. indexOf判断参数值的第一个出现位置，所以去比较是否===-1

  2. indexOf无法识别NaN，includs可以判断

entries,keys,values

都可以用for of来循环遍历

flat

数组降维

  1. 不传参数时只会拉平一层

  2. 参数为n时，可以拉平n层

  3. 参数为Infinity时，不管多少层嵌套，都可以转成一维数组

flatMap

对原数组的每个成员执行一个函数，然后对返回值执行flat方法返回一个新数组，不会改变原数组

注：
flatMap()只能展开一层数组

总结

  1. Array.from与Array.of属于构造函数

  2. entries、keys、values、find、findIndex属于数组遍历方法

  3. copyWithin和fill会改变数组自身
